# providerQuotas Plan Usage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove the legacy `limits` and `usageLimits` state/settings shapes and make `providerQuotas` the single quota state source for Plan Usage, the floating widget, tray quota labels, and alerts.

**Architecture:** `StateManager` publishes effective provider quota snapshots keyed by provider. Renderer components consume shared quota presentation selectors that map machine-readable quota windows and status into Rich/Simple/None display models plus API/Cache/Bridge/Log badges. API/Cache is a derived presentation badge from `ProviderQuotaWindow.source`, not its own app-state field. Plan Usage no longer accepts local token-activity burn-rate ETA as a quota source; local provider totals remain available as statistics paired with provider quota snapshots.

**Tech Stack:** Electron main process, React renderer, TypeScript, Node test runner.
